# Nature du code, sources et usage de l'IA

## Degré de création originale

Le projet PhishGuard a été conçu, structuré, développé et documenté dans le cadre du travail de l'équipe.
L'architecture générale, l'organisation du code, la logique de traitement et la documentation finale relèvent d'un travail de conception du projet.

## Sources utilisées

Nous avons utilisé des bibliothèques Python externes pour des briques techniques standard, notamment pour :
- l'API ;
- le parsing de fichiers email ;
- certaines validations ;
- le lancement du serveur web.

Les dépendances nécessaires au projet sont listées dans `requirements.txt`.

Nous avons également consulté de la documentation technique pour comprendre le format des emails, le fonctionnement de FastAPI et certains aspects liés au phishing.

## Réutilisation éventuelle

Le projet n'est pas une copie d'une application existante. Il peut s'appuyer sur des idées, des conventions ou des exemples techniques très courts, mais la structuration du projet et l'intégration finale ont été réalisées pour ce travail.

## Usage de l'intelligence artificielle

L'intelligence artificielle a été utilisée de manière limitée, réfléchie et relue.

Elle a surtout servi à :
- reformuler certains textes ;
- suggérer des améliorations de documentation ;
- aider à clarifier certaines explications techniques.

L'IA n'a pas remplacé le travail de compréhension, de validation, de test ni les décisions techniques prises sur le projet.
Toutes les propositions conservées ont été relues, comprises et adaptées.
